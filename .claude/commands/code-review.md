# Code Review

現在の未コミット変更に対して包括的なコードレビューを実施してください。

## レビュー手順

1. `git diff` でステージングされた変更とされていない変更を確認
2. 以下の観点でレビュー:

### チェック項目
- **データ整合性**: 計算ロジックが baseAmount から再計算しているか、ゼロ除算ガード
- **パターン準拠**: factory コンストラクタ、copyWith、Riverpod Provider パターン
- **エラーハンドリング**: 非同期処理の例外処理、null 安全性
- **コード品質**: 関数サイズ、ネスト深度、不要な print()
- **UI**: 日本語テキスト、Material 3 準拠

3. 問題をCRITICAL/HIGH/MEDIUM/LOWで分類して報告
4. 各問題に具体的な修正案を提示

## 出力形式

```
### [重要度] — [概要]
**ファイル**: `path:行番号`
**問題**: ...
**修正案**: ...
```
