# エージェントオーケストレーション

## 利用可能なエージェント

| エージェント | 目的 | 使用タイミング |
|-------------|------|---------------|
| planner | 実装計画 | 複雑な機能追加、リファクタリング |
| code-reviewer | コードレビュー | コード変更後 |
| build-error-resolver | ビルドエラー修正 | flutter analyze / build 失敗時 |
| tdd-guide | テスト駆動開発 | 新機能、バグ修正 |
| doc-updater | ドキュメント更新 | コード変更に伴うドキュメント同期 |

## 即時使用の推奨

ユーザーの指示を待たずに使用:
1. 複雑な機能リクエスト → **planner**
2. コードを書いた直後 → **code-reviewer**
3. バグ修正や新機能 → **tdd-guide**
4. ビルド失敗 → **build-error-resolver**

## 並列実行

独立した操作には並列 Task 実行を使用:

```
# 良い例: 並列実行
1. セキュリティレビュー（code-reviewer）
2. ドキュメント更新確認（doc-updater）

# 悪い例: 不要な順次実行
1つずつ実行
```
